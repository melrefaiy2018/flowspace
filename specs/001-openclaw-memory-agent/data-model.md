# Data Model: OpenClaw Memory Agent

## Entities

### StagedDraft

A meeting prep brief generated by the horizon scanner.

```typescript
interface StagedDraft {
  id: string;                // uuid (generated at scan time)
  meetingId: string;         // Google Calendar event ID (dedup key)
  meetingTitle: string;
  meetingTime: string;       // ISO 8601
  attendees: string[];       // email addresses
  summary: string;           // LLM-generated brief (markdown, max 500 words)
  linkedDocs: LinkedDoc[];
  relatedEmails: RelatedEmail[];
  suggestedActions: string[];
  confidence: number;        // Phase 1: always 1.0. Phase 2: 0.0-1.0
  confidenceReason: string;  // Phase 1: "". Phase 2: e.g. "2 prior approvals"
  useful?: boolean;          // Phase 1 toggle. Phase 2: replaced by signals.
  createdAt: string;         // ISO 8601
  seenAt?: string;           // Set when GET /api/drafts is called
  status: DraftStatus;
}

type DraftStatus = "pending" | "approved" | "dismissed" | "error";

interface LinkedDoc {
  title: string;
  url: string;
}

interface RelatedEmail {
  subject: string;
  from: string;
  snippet: string;
}
```

**Storage**: `DATA_DIR/staged-drafts.json` via SharedJsonFileStore.
**Lifecycle**: Created by scan. Updated by approve/dismiss/useful. Auto-purged after 7 days or past meetingTime.
**Dedup**: Upserted by `meetingId` on scan (same meeting = update, not duplicate).

### ScanResult

Output of a horizon scan.

```typescript
interface ScanResult {
  drafts: StagedDraft[];
  meta: ScanMeta;
}

interface ScanMeta {
  scannedAt: string;        // ISO 8601
  meetingsFound: number;    // Total qualifying meetings
  meetingsPrepped: number;  // Successfully briefed
  errors: ScanError[];      // Per-meeting failures
}

interface ScanError {
  meetingId: string;
  meetingTitle: string;
  error: string;            // Human-readable error message
}
```

### StagedDraftsFile

On-disk format for staged-drafts.json.

```typescript
interface StagedDraftsFile {
  version: 1;
  drafts: StagedDraft[];
  lastScan?: ScanMeta;      // Most recent scan metadata
}
```

## Relationships

```
ScanResult 1──* StagedDraft    (one scan produces many drafts)
ScanResult 1──1 ScanMeta       (one scan has one metadata record)
ScanMeta   1──* ScanError      (one scan may have many errors)
StagedDraft *──1 CalendarEvent  (many drafts reference one meeting, dedup by meetingId)
```

## State Transitions

```
StagedDraft.status:

  [created by scan]
       │
       ▼
   "pending"
       │
       ├──── approve ────→ "approved"  (opens chat with threadBrief)
       │
       ├──── dismiss ────→ "dismissed"
       │
       └──── auto-purge ──→ [deleted]  (>7 days or past meetingTime)

  [scan error]
       │
       ▼
    "error"  (displayed in UI as error state)
```

## Validation Rules

| Field | Rule |
|-------|------|
| `id` | Non-empty UUID string |
| `meetingId` | Non-empty string (Google Calendar event ID) |
| `meetingTime` | Valid ISO 8601 datetime |
| `attendees` | Non-empty array of email strings |
| `summary` | Non-empty string, max 500 words |
| `confidence` | Number 0.0-1.0 |
| `status` | One of: "pending", "approved", "dismissed", "error" |
| `createdAt` | Valid ISO 8601 datetime |

## Phase 2 Extensions

### ImportanceSignal (separate file, not new MemoryCategory)

```typescript
// DATA_DIR/.importance-signals.json
interface ImportanceSignal {
  entityId: string;         // attendee email | meeting type keyword | domain
  signal: "approved" | "dismissed" | "edited" | "ignored";
  weight: number;           // bounded: -10.0 to +10.0
  context: string;          // what was prepped
  updatedAt: string;        // ISO 8601
}
```

- Weight decay: halved every 30 days
- Skip threshold: weight < -3.0
- Bootstrap from Phase 1 `useful` booleans: true → +1.0, false → -0.5
