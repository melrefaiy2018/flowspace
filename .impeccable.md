# FlowSpace ‚Äî Impeccable Design Context

## Design Context

### Users
Technical professionals ‚Äî developers, PMs, and ops folks who live in terminals, dashboards, and command-line tools. They use FlowSpace as part of their daily work rhythm: checking what needs attention, acting on email and calendar, and querying their workspace via AI. They expect **density, precision, and speed** ‚Äî not hand-holding. The interface should feel like a tool built by someone who uses it, not a product designed to impress in a demo.

### Brand Personality
**Premium productivity tool** ‚Äî polished like Linear or Vercel, with hints of Arc's personality. Every detail earns its place. The product should feel worth paying for even though it's free to self-host. The green accent (`#22c55e`) is the single source of life in an otherwise near-black canvas.

Three-word personality: **Precise. Ambient. Confident.**

Emotional goals: Users should feel *in control*, *informed without overwhelm*, and *trusted by their tools*.

### Aesthetic Direction
- **Theme**: Dark-only. Deep near-blacks (`#1b1b1d` ‚Üí `#070708`) layered with subtle surface elevation.
- **Accent**: Green-500 (`#22c55e`) used sparingly ‚Äî live indicators, CTAs, accent bars, active states. Never decorative.
- **Typography**: Geist Sans for UI text with tight letter-spacing (`-0.01em`). Geist Mono for kickers, timestamps, metadata, and code. Mono uppercase with wide tracking (`0.16em`) for section labels is a signature pattern ‚Äî preserve it.
- **References**: Linear (typographic density, dark precision), Vercel dashboard (subtle gradients, clean data hierarchy), Arc Browser (personality within dark constraints ‚Äî slightly warmer, opinionated details).
- **Anti-references**: Avoid anything that feels generic SaaS (rounded everything, blue primary, white backgrounds). Avoid busyness ‚Äî no competing focal points. Avoid "app-ified" patterns ‚Äî this is closer to a power tool than a consumer app.
- **Border radius**: Mixed and intentional ‚Äî small chips at 7px, interactive cards at 12-16px, hero panels at 24-30px. The scale expresses hierarchy.
- **Motion**: Restrained. Pulse on live indicators, subtle fade/slide for panels, slow spin on loading. Never decorative animation.

### Design Principles

1. **Density over whitespace** ‚Äî Technical users read fast. Pack information tightly, but use typographic weight and color (not spacing) to create hierarchy.

2. **Green is signal, not style** ‚Äî `#22c55e` appears only where it means something: live state, primary action, progress, success. Overusing it kills its power.

3. **Surfaces tell a story** ‚Äî The layered dark palette (`surface` ‚Üí `surface2` ‚Üí `surface3` ‚Üí elevated) is how depth is communicated. Every card and panel should sit at the right layer.

4. **Mono text earns its place** ‚Äî Geist Mono is reserved for machine-generated or structured content: timestamps, kickers, metadata, shell-like labels. Using it for prose breaks the contract.

5. **Motion should feel earned** ‚Äî Animate only state changes that would otherwise feel abrupt. Prefer opacity and translate over scale. Duration under 200ms for micro-interactions.

## Token Reference (from `src/index.css`)

| Token | Value | Use |
|-------|-------|-----|
| `--bg` | `#1b1b1d` | Base page background |
| `--bg-elevated` | `#070708` | Deepest layer (sidebar, modals) |
| `--surface` ‚Üí `--surface3` | `#1f2023` ‚Üí `#2d2e33` | Card elevation scale |
| `--accent` | `#22c55e` | Primary green ‚Äî use sparingly |
| `--accent-dim` | `#0d2818` | Muted green backgrounds |
| `--text` | `#ededed` | Primary text |
| `--text-dim` | `#b4b4bb` | Secondary text |
| `--text-faint` | `#7d7d86` | Meta, placeholders, kickers |
| `--border` | `#2b2c31` | Default borders |
| `--radius-sm/md/lg` | `7px / 12px / 16px` | Standard radius scale |

## Focus Areas for Impeccable Commands

- `/audit` ‚Äî Check for typographic inconsistencies, off-scale radii, accent overuse, missing hover states
- `/normalize` ‚Äî Align spacing, font sizes, and border radius to the established token scale
- `/polish` ‚Äî Refine motion, micro-interactions, and edge states (loading, empty, error)
- `/distill` ‚Äî Remove visual noise; anything that doesn't serve the user's task should go
